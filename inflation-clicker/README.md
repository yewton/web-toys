# インフレクリッカー (Inflation Clicker)

桁を上げて、天文学的な敵 HP を削り切るクリッカー。
削れていくキングダムハーツ風の分割体力ゲージが中心。

## 遊び方

1. コースを選んでスタート（時間コース 1分/5分/30分 ＋ 極限モード 2つ）
2. 敵（👿）を連打して攻撃する
3. たまに出現する ⚔️ アイテムを取ると攻撃力（の桁）が伸びる
4. 攻撃力の桁が敵 HP の桁に追いつけば撃破

進行状況は `localStorage`（`inflationClicker.save`）へ自動保存され、メニューの **つづきから** で再開できる。外部サービスへの依存はない。

## 数の表示

巨大数は仮数 × 10^指数（`BigNum`）で保持する。万進法の範囲（〜10^72 弱、無量大数まで）は**複数単位を連結した複合漢数字**で表示する（例: `123極4500載67正`）。無量大数（10⁶⁸）より上は、**華厳経「上数法」の命数**（矜羯羅 10¹¹²、阿伽羅 10²²⁴、最勝 10⁴⁴⁸、摩婆羅、阿婆羅、多婆羅、界分… 各々が前の「2乗」）で**割って商＋命数名**で表す。例: `10^112` → `1矜羯羅`、`10^150` → `100澗矜羯羅`、`10^224` → `1阿伽羅`、`10^448` → `1最勝`。これにより**登るにつれて新しい漢字（命数）が次々現れる**。命数が倍々間隔なので、各命数レンジの上の方では商が大きくなり無量大数が係数に混じることがある（例: `1垓無量大数矜羯羅`）。命数の無い帯（10⁷²〜10¹¹¹）は無量大数を超単位に積み上げ（`1万無量大数`）。命数・積み上げの範囲も超える（グラハム数級など）場合は**指数を漢数字にした冪乗の塔**にフォールバック（例: `10^10^308`）。

ダメージ数値の下には**科学表記を小さく添える**（例: `8載8443正…無量大数` の下に `8.8443 × 10^112`）。馴染みのない命数でも大きさが一目で分かる。攻撃力表示も漢数字（大）＋科学表記（小）の二段。画面に出す数字は伸びていく**攻撃力**のみ（HP はゲージで表現し、数値表示はしない）。

> 補足: `不可説不可説転` には複数定義がある（華厳経テキストでは 10^45 程度）。本作はコース/モードの大小（無量大数 &lt; 不可説不可説転 &lt; グラハム数）を保つため、大数の一覧で一般的な `10^(7×2¹²²) ≈ 10^(3.7×10³⁷)` を採用している。

指数そのものが巨大になる極限モード（不可説不可説転級など）では、**指数も `a×10^b` 表記に圧縮**して横幅の暴走を防ぐ（例: `1×10^1.00×10^18`）。それでも長い場合、ヘッダー/フッターの数値は**枠幅に合わせて自動縮小**（`fitText`）し、飛び出すダメージ数値（複合漢数字は最大 3 単位で横長になりうる）は**画面幅に収まるようフォント縮小＋中央寄せ**して、桁が巨大でも全体が必ず見えるようにする（雰囲気優先）。

## モバイル / パフォーマンス

- レイアウトは `dvh`（実表示高さ）ベースの flex。`header`(固定) / `main`(伸縮) / `footer`(固定) で、ブラウザ UI が出ていてもフッターが必ず収まる。
- 連打中のズーム暴発を防止：`body` は `touch-action: manipulation`、連打ゾーン `#arena` は `touch-action: none`（ダブルタップズーム無効）。iOS Safari は `user-scalable=no` を無視するため `gesturestart`/`gesturechange` を抑止（＝ピンチズーム無効。Android はビューポートメタで抑止）。
- 描画ループは**ダーティチェック**で無駄を省く：ステータス DOM は値が変わったときだけ書き換え、ゲージ Canvas は位置・赤残像・箱点滅のいずれかが動くときだけ再描画、ダメージ用フルスクリーン Canvas はパーティクルが居るときだけ更新。大きな絵文字やパネルに**連続 CSS アニメは置かない**（硬化オーラ・発光は静的、変化時のみ更新）。
- ダメージ数値パーティクルは**寿命を実時間基準（約 0.8 秒）**にし（fps が落ちても滞留時間が延びない）、個数上限 36・1 行のみ描画・フルスクリーン Canvas の DPR は 2 上限に抑える。
- 4× CPU スロットル下でも、最も横長な複合漢数字を毎フレーム連打しても **60fps を維持**。

## 体力ゲージ（KH 風・バー消化＋箱コンベア）

全コース共通で、HP を「**1 本＝固定の指数幅**で削れるバー」の積み重ねとして扱う。総バー数 = HP指数 / `GAUGE.BAR_E`。表示する箱は **可視窓**（コース固有の `segments−1` 本）まで。

| 要素 | 表現 |
|---|---|
| 削れる向き | **左から削れる**（フィルは右寄せ、空き＝黒が左に伸びる）。1 本削り切ると即・満タンの次の本へ |
| 最前面のバー | いま削っている 1 本。基本は緑、黒背景。**最後のバーだけ多層**（緑→青→黄→黒のめくり） |
| 最後のバーの長さ | 総HP指数が BAR_E の整数倍でないコース（1分・5分など）では、最後のバーは BAR_E より短い。表示は実長にスケールして「最後のバーが完走しきった瞬間」に撃破がくる |
| コンボ中の赤 | 直近ダメージ分が赤い残像としてフィルの左側に伸びる（攻撃が止まると追従して消える） |
| バー下の箱（**補充モード**） | 残りバーが窓より多いとき（大コース・極限）。右寄せの窓 `segments−1` 個が常に満杯で、1 本削り切ると左端が赤→黒→左へスライドしてフェードアウトし、残りが左へ詰めて、**右から新箱がフェードイン**＝コンベア |
| バー下の箱（**枯渇モード**） | 残りバーが窓以下になったとき（時間コース終盤＝climax）。箱はその場に右寄せのまま据え置き、1 本削り切ると **箱の 1 つ左に退場用の箱が現れて赤→黒→左へスライドして消える**。新箱は補充されず、左から尽きていく |
| 撃破 | `atk.e ≥ maxHp.e`（残り 0 本）で撃破。時間コースは到達、極限コースは事実上不可 |

## コース

全コース**共通の「バー消化＋箱コンベア」型ゲージ**で削る。1 本のバーは固定の指数幅（`GAUGE.BAR_E`）で削れ、削り切ったら箱が 1 つ消えて次のバーが現れる。**1 本目の削れペースは全コースで同じ**。コースの大きさは**箱の総本数**で表す（総本数 = HP指数 / BAR_E）。さらに**1 本ごとに段々硬く**なる（`barHardening` は全コース同じカーブ）。

| コース | 総HP | 総バー数 | 見える箱の最大本数 | 撃破時の命数 |
|---|---|---|---|---|
| 1分 | 10⁶⁸ | 約 8 本 | 7 | 無量大数 |
| 5分 | 10³⁴⁰ | 約 42 本 | 11 | 阿伽羅 |
| 30分 | 10²⁰⁴⁰ | 約 255 本 | 15 | 阿婆羅 |

**極限モード（自己責任）** — 総バー数が桁外れに多く、しかも同じ硬化カーブを当てるので奥は限りなく硬くなる＝事実上クリア不能。巨大数の途方もなさを味わう枠（メニューでは琥珀色の警告カード）。前面バーは硬化の floor (`BAR_HARDEN_FLOOR = 0.06`) でゆっくりだが進み続け、箱はコンベアで右から補充され続ける。

| モード | 総HP | 総バー数 | 毎秒10クリックでの目安 |
|---|---|---|---|
| 不可説不可説転級 | 10^(3.7×10³⁷) | 約 4.6×10³⁶ 本 | 約 10³⁰ 年 |
| グラハム数級 | 10^(1.7×10³⁰⁸) | 約 2.1×10³⁰⁷ 本 | 約 10³⁰⁰ 年（宇宙年齢 ≈ 1.4×10¹⁰ 年） |

## バランス・演出

ゲージは**攻撃力の指数で直接駆動**する（`progress = atk.e / BAR_E` が消化バー数、`atk.e ≥ maxHp.e` で撃破）。log(累計ダメージ) を使わないので、終盤に累計が加速して一気に崩れる、という挙動が起きない。

- 実際の攻撃力 `atk.e` は目標指数へ**毎クリック近づく**（差に比例しない＝パワーアップ直後でも一気に削れない）。基本歩幅 = `maxHpLog / (totalItems × CLICK_BUDGET_PER_ITEM)`
- **硬度カーブ（1 本ごと・全コース共通）**：歩幅に `barHardening(barsChipped) = BAR_HARDEN_FLOOR + (1 − BAR_HARDEN_FLOOR) / (1 + 削り切ったバー数 / BAR_HARDEN_SCALE)` を掛ける。1 本目は ×1（サクサク）、削るほど 1 クリックの伸びが鈍り、奥は ×`BAR_HARDEN_FLOOR` 付近まで落ちる（**完全停止＝フリーズを避けるため floor は 0 にしない**）。どのコースも同じカーブなので「序盤の 1 本目は同ペース、奥へ行くほど硬い」。時間コースは終盤がクライマックス、極限コースはインフレの果てに floor 付近に張り付く
- **終盤は純粋な粘り勝負**：残り箱が窓（=可視最大本数）以下になったらアイテムは出ず、攻撃力目標も解放済み。時間コースの最後の数本がノーアイテムの climax になる（極限コースでは遠未来に発火するので実質発火しない）
- アイテムは**燃料補給式**：攻撃力が目標へあと 1 アイテム分弱まで詰まったら自動で出る（終盤手前まで）。序盤は頻繁・終盤は間遠に出て**パワーアップが切れず**、攻撃力は常に目標を追う（プラトーなし）。取ると目標指数が 1 アイテム分上がる
- **燃料切れでもゲージは止まらない**：目標に追いついた状態でも 1 クリックで `IDLE_TRICKLE_RATIO`（通常歩幅の 1 割）ぶんだけ攻撃力と目標が進むので、バーは常に少しずつ削れる。アイテム無しでも“非現実的な手数”をかければ撃破でき、アイテムを取れば桁違いに速い（＝アイテムが主役のまま、完全な壁にはならない）
- ダメージは「一桁ランダム × 攻撃力の桁」。初期は 1〜9、アイテムで確実に桁が増える
- アイテム出現はクリック数ベース（約 7〜11 クリックに 1 個）

演出：叩くたびに**ダメージ数値**（読みやすさ優先で小さめ・短め＝上位2単位・大きく拡散）＋着弾点の控えめなインパクトリング／スパーク、敵が**一瞬縮んで傾くリコイル**（`--hit` を毎フレーム減衰、transform のみで軽量）。硬化は文字ではなく、削れるほど敵に**結晶シェル（二重のエネルギーリング）**が濃く重なり、叩いても**怯みにくく**なることで視覚的に示す。攻撃力ランプ中は攻撃表示が発光。アイテム（⚔️ ＝攻撃力アップ）取得時は文字ではなく**バフ演出**：取得地点からエメラルドの光が立ち上り（上向きスパーク）＋波紋リング、攻撃パネルが一瞬強く発光する。撃破時は**敵が弾け飛ぶアニメ＋画面フラッシュ＋スパーク爆発**を約 0.9 秒見せてから結果画面に切り替える。

結果画面の「タイトルへ」ボタンは、連打の勢いで即タイトルに戻らないよう **画面下端（連打ゾーンの中央から離れた位置）** に置き、さらに表示直後の約 0.7 秒は非アクティブ（武装解除）にしてある。

1 本目はどのコースも約 21 クリック（共通歩幅）。1分コースは 8 本でクリア（終盤の数本がやや重い）。長いコースは本数が増えるだけでなく深いバーほど硬くなるので、想定時間はおよそ「コース名そのもの」のペース。チューニングは `config.ts` の `GAUGE.BAR_E`（1 本あたりの指数幅）と `ATTACK.BAR_HARDEN_FLOOR`／`BAR_HARDEN_SCALE`（硬化カーブ）、`CLICK_BUDGET_PER_ITEM`（共通歩幅）、`CHUNK_E`（1 アイテム分）。

## 構成

```
inflation-clicker/
├── index.html        画面の骨組み（メニュー / ゲーム / クリア演出）
└── src/
    ├── main.ts        エントリポイント
    ├── bignum.ts      巨大数クラス BigNum（仮数 × 10^指数）— 純粋ロジック
    ├── format.ts      BigNum → 複合漢数字 / 英語 / 科学表記 / mm:ss — 純粋ロジック
    ├── config.ts      難易度設定・攻撃/ゲージのチューニング値・配色
    ├── state.ts       ゲーム状態シングルトン＋localStorage セーブ/ロード
    ├── hpGauge.ts      バー消化＋箱コンベアのモデル（progress / 残像 / 箱退場検知 / barHardening）— 純粋ロジック
    ├── gaugeView.ts    ゲージの Canvas 描画（モデルは hpGauge.ts）
    ├── particles.ts   ダメージ数値パーティクル（物理＋Canvas 描画）
    ├── ui.ts          DOM 配線（メニュー生成・画面切替・ステータス更新・入力）
    └── game.ts        ゲームロジックと描画ループ（攻撃 / アイテム / セーブ）
```

描画とロジックは分離してある。ゲージの位置計算・残像追従・箱の削れ判定は `hpGauge.ts` に純粋関数として置き、Canvas 依存の描画だけ `gaugeView.ts`（カバレッジ対象外）に切り出している。
