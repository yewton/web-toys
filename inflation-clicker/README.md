# インフレクリッカー (Inflation Clicker)

桁を上げて、天文学的な敵 HP を削り切るクリッカー。
削れていくキングダムハーツ風の分割体力ゲージが中心。

## 遊び方

1. コースを選んでスタート（到達命数で名付けた 5 コース。下 2 つは極限）
2. 敵を連打して攻撃する
3. たまに出現する ⚔️ アイテムを取ると攻撃力（の桁）が伸びる
4. 累積ダメージの桁が敵 HP の桁に追いつけば撃破

進行状況は **コースごとに** `localStorage`（`inflationClicker.save.<コース>`）へ自動保存され、メニューの **つづきから** で再開できる。外部サービスへの依存はない。

## コース

コース間の差異は **敵の総 HP（指数）だけ**。攻撃・ゲージ・密度のチューニングは全コース共通で、1 本目の削れペースもアイテム頻度も同じになるよう設計されている（`config.ts` の動的密度。詳細は後述／rules）。各コースは到達する HP 指数の命数で命名している。

| コース | 総HP | 総箱数(≈) | 表示 | 目安 |
|---|---|---|---|---|
| 無量大数 | 10⁶⁸ | 7 | そのまま並ぶ | サクッと 1 分 |
| 摩婆羅 | 10⁸⁹⁶ | 32 | 表示上限ちょうど | そこそこ 5 分 |
| 界分 | 10⁷¹⁶⁸ | 53 | コンベア（上限超） | じっくり 8 分 |
| 不可説不可説転 ⚠️ | 10^(7×2¹²²) ≈ 10³·⁷ˣ¹⁰³⁷ | 約 830 | コンベア | 極限 2 時間 |
| グラハム数 ⚠️ | 10^(1.7×10³⁰⁸) | 約 7000 | コンベア | 連打で 1 週間級 |

下 2 つ（⚠️）は `extreme` フラグ付きで、メニューでは琥珀色の警告カードとして描く。事実上クリア不能ではないが、文字通り張り付くプレイになる枠。巨大数の途方もなさを味わうためのもの。

> 補足: `不可説不可説転` には複数定義がある（華厳経テキストでは 10⁴⁵ 程度）。本作はコースの大小（無量大数 &lt; 不可説不可説転 &lt; グラハム数）を保つため、大数一覧で一般的な `10^(7×2¹²²)` を採用している（華厳経「上数法」の命数 n=122 にちょうど一致）。

## 数の表示

巨大数は仮数 × 10^指数（`BigNum`）で保持し、桁の帯ごとに表記を切り替える（`format.ts`）。

- **〜10⁷²弱（万進法）**: 複数単位を連結した複合漢数字（例: `123極4500載67正`）
- **10¹¹²以上（華厳経「上数法」命数）**: 矜羯羅 10¹¹²、阿伽羅 10²²⁴、最勝 10⁴⁴⁸… **各々が前の 2 乗**。最大の命数で割った商＋命数名で表し、商はさらに再帰整形して小さい命数が大きい命数に入れ子になる（例: `10¹¹²`→`1矜羯羅`、`10²²⁴`→`1阿伽羅`、`10⁸⁰⁰`→`1京矜羯羅阿伽羅最勝`）。登るにつれ新しい漢字（命数）が次々現れる
- **命数の無い帯（10⁷²〜10¹¹¹）**: 無量大数（10⁶⁸）を超単位に積み上げ（例: `1万無量大数`）
- **命数・積み上げも超える（グラハム数級）**: 指数を漢数字化した冪乗の塔にフォールバック（例: `10^10^308`）

ダメージ数値の下には**科学表記を小さく添える**（例: `8.8443 × 10^112`）。馴染みのない命数でも大きさが一目で分かる。命数名には**読み仮名（ルビ）**が付く。画面に出す数字は伸びていく**攻撃力**のみ（HP はゲージで表現し、数値表示はしない）。

指数そのものが巨大になる極限コースでは、**指数も `a×10^b` 表記に圧縮**して横幅の暴走を防ぐ（例: 不可説不可説転コースの HP は `1 × 10^3.70×10^37`）。それでも長い場合は、ヘッダー/フッターの数値を**枠幅に合わせて自動縮小**（`fitText`）し、飛び出すダメージ数値は**画面幅に収まるようフォント縮小＋中央寄せ**する。

> 表記ポリシー（`format.ts`／rules）: 厳密でなくてよいが嘘にはしない／命数で数字の羅列を避ける／無量大数を無尽蔵に並べない。表記の変更時はこの 3 つを必ず保つこと。

## 体力ゲージ（KH 風・バー消化＋箱コンベア）

全コース共通で、HP を「**1 本ずつ削れるバー**」の積み重ねとして扱う。総箱数（segments）と削れペースは固定の指数幅ではなく、**動的密度**から導く（`consumedFromDamageE` の積分。コース全域で per-click の削れ感が一定になる ＝ HP しか変えないという設計を成立させる仕組み）。表示する箱は**上限 32**（`DISPLAY_CAP_BOXES`）まで。

| 要素 | 表現 |
|---|---|
| 削れる向き | **左から削れる**（フィルは右寄せ、空き＝黒が左に伸びる）。1 本削り切ると即・満タンの次の本へ |
| 最前面のバー | いま削っている 1 本。基本は緑、黒背景。**最後のバーだけ多層**（緑→青→黄→黒のめくり） |
| コンボ中の赤 | 直近ダメージ分が赤い残像としてフィルの左側に伸びる（攻撃が止まると追従して消える） |
| バー下の箱（**コンベアモード**） | 総箱数が表示上限を超えるコース（界分以上）。右寄せの窓が常に満杯で、1 本削り切ると左端が赤→黒→左へスライドしてフェードアウトし、**右から新箱がフェードイン** |
| バー下の箱（**枯渇モード**） | 総箱数が表示上限以下のコース（無量大数・摩婆羅）の終盤。箱は据え置きで、左から尽きていく |
| 撃破 | 累積ダメージ指数 `damageE ≥ hp.e`（残り 0 本）で撃破 |

## バランス・演出

ゲームプレイは **3 変数・フェーズレス**（`mechanics.ts`／`state.ts`）。フェーズゲートや燃料・アイドル進行はない。

- `atk.e` — 現在の 1 クリックダメージの桁。**アイテム取得後のランプでのみ**動く
- `atkTargetE` — ランプの目標。アイテム 1 個ごとに `chunkAtE` 分ジャンプする
- `damageE` — 累積ダメージ（log10）。`damageE ≥ hp.e` で撃破

挙動の要点:

- **攻撃力はアイテム取得後だけ伸びる**: 取得で `atkTargetE` が 1 アイテム分上がり、以後 1 クリックごとに `chunkAtE(atk.e) / CLICK_BUDGET_PER_ITEM` の歩幅で目標へ追従する（取得直後に一気に削れず滑らかに伸びる）。追いつくと止まる ＝ 次のアイテムまで桁は据え置き
- **アイテムはクリック数ペース**: 取得から `CLICK_BUDGET_PER_ITEM = 9` クリックで次の 1 個が出る（`totalItems` 上限まで）。序盤頻繁・終盤間遠といった動的調整はせず、純粋にクリック数で出る
- **動的密度でコース不問の削れ感**: `chunkAtE(e) = max(1, e/100)`、`expPerBoxAt = 10 × chunkAtE`。両者が連動して伸びるので、`INFL_E = 100`（無量大数のすぐ上）から上では指数が上がってもゲージの per-click 削り量はほぼ一定。「無量大数を越えた瞬間から世界が加速する」物語に対応
- **硬化は見た目だけ**: `setHardening` は HP 残量割合（`damageE / hp.e`）で敵の結晶シェルを濃くするだけのコスメ。ゲームプレイ上の硬化（歩幅を鈍らせる等）は**無い**。極限コースでは割合がほぼ 0 のままなのでシェルもほぼ出ない
- ダメージは「一桁ランダム × 攻撃力の桁」。初期は 1〜9、アイテムで確実に桁が増える

演出: 叩くたびに**ダメージ数値**（上位 2 単位・大きく拡散）＋着弾点の控えめなインパクトリング／スパーク、敵が**一瞬縮んで傾くリコイル**。硬化は文字ではなく敵に重なる**結晶シェル**で示す。アイテム取得時は文字ではなく**バフ演出**（エメラルドの光＋波紋リング＋攻撃パネルの強発光）。撃破時は**敵が弾け飛ぶアニメ＋画面フラッシュ＋スパーク爆発**（KH 風スローモーション、約 5 秒）を見せてから結果画面に切り替える。結果画面の「タイトルへ」ボタンは連打の勢いで即戻らないよう画面下端に置き、表示直後の約 0.7 秒は非アクティブにしてある。

チューニングは `config.ts` の `GAUGE_DENSITY`（`ITEMS_PER_BOX` / `C0` / `INFL_E`）、`DISPLAY_CAP_BOXES`、`ATTACK.CLICK_BUDGET_PER_ITEM` に集約。コースを足すときは `course(name, desc, hpE, extreme)` を呼ぶだけ（HP 以外は触らない）。

## モバイル / パフォーマンス

- レイアウトは `dvh`（実表示高さ）ベースの flex。`header`(固定) / `main`(伸縮) / `footer`(固定) で、ブラウザ UI が出ていてもフッターが必ず収まる
- 連打中のズーム暴発を防止: `touch-action` ＋ iOS Safari 向けの `gesturestart`/`gesturechange` 抑止（ピンチズーム無効）
- 描画ループは**ダーティチェック**: ステータス DOM は値が変わったときだけ、ゲージ Canvas は位置・赤残像・箱演出が動くときだけ、ダメージ用フルスクリーン Canvas はパーティクルが居るときだけ更新。大きな絵文字やパネルに連続 CSS アニメは置かない
- ダメージ数値パーティクルは**寿命を実時間基準**（fps が落ちても滞留時間が延びない）にし、個数上限・1 行描画・フルスクリーン Canvas の DPR 2 上限で抑える
- 4× CPU スロットル下でも、最も横長な複合漢数字を毎フレーム連打しても **60fps を維持**

## 構成

```
inflation-clicker/
├── index.html        画面の骨組み（メニュー / ゲーム / クリア演出）
└── src/
    ├── main.ts        エントリポイント
    ├── bignum.ts      巨大数クラス BigNum（仮数 × 10^指数）— 純粋ロジック
    ├── format.ts      BigNum → 複合漢数字 / 英語 / 科学表記 / ルビ / mm:ss — 純粋ロジック
    ├── config.ts      コース定義・動的密度・攻撃/ゲージのチューニング値・配色
    ├── state.ts       ゲーム状態シングルトン＋難易度別 localStorage セーブ/ロード
    ├── mechanics.ts   1 クリック / 1 アイテム分の状態遷移（攻撃・ダメージ加算・アイテム供給）— 純粋ロジック
    ├── hpGauge.ts     バー消化＋箱コンベアのモデル（progress / 残像 / 箱退場検知）— 純粋ロジック
    ├── gaugeView.ts   ゲージの Canvas 描画（モデルは hpGauge.ts。カバレッジ対象外）
    ├── particles.ts   ダメージ数値パーティクル（物理＋Canvas 描画・ルビ）
    ├── ui.ts          DOM 配線（メニュー生成・画面切替・ステータス更新・入力）
    └── game.ts        ゲームロジックと描画ループ（攻撃 / アイテム / セーブ / 撃破演出）
```

描画とロジックは分離してある。状態遷移は `mechanics.ts`、ゲージ計算は `hpGauge.ts` に純粋関数として置き、Canvas 依存の描画だけ `gaugeView.ts` に切り出している。**ファイル横断の不変条件や非自明な内部仕様は [`.claude/rules/inflation-clicker.md`](../.claude/rules/inflation-clicker.md) に集約**してあるので、各 `src` を触る前にそちらを参照。リポジトリ全体のコマンド・CI 等は [`../CLAUDE.md`](../CLAUDE.md)。
