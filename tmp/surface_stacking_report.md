# 地上スタック問題 調査レポート

## 問題の概要

仮説C（右上滞留バグ）を修正した後、新たに「アリが画面最上端（y ≈ 0）にスタックして動けなくなる」問題が発生した。
300,000 ステップ実行後に 60〜88% のアリが `y < 5` に集積する。

---

## 発生の経緯

| 修正試行 | `avoidObstacle` の変更内容 | 300k 時点の top_edge |
|---|---|---|
| 修正なし（元のバグ） | 非対称リダイレクト（右偏り） | 右上に 100% 集積 |
| 試行1（hypo-C ブランチ）| ランダムターン ±60°〜±120° | **88%** が y<5 に集積 |
| 試行2 | 対称水平リダイレクト（左右ランダム） | **66%** が y<5 に集積 |
| 試行3（現在） | 試行2 + wanderAngle バグ修正中 | 未検証 |

---

## 特定された根本原因（複数）

### 原因A: `avoidObstacle` の方向リダイレクトの非対称性（仮説Cの本体）

**該当箇所:** `Ant.ts` `avoidObstacle()` 旧コード

```ts
this.angle = Math.cos(this.angle) > 0 ? -0.2 : Math.PI + 0.2;
```

- 右向きアリ → `-0.2`（右水平・わずかに上）
- 左向きアリ → `π + 0.2`（左水平・わずかに下）

右向きアリが地表に押し上げられ続け、300k ステップで全員が右端に集積。
**修正方針:** 左右をランダムに選ぶ対称水平リダイレクトに変更。ただし角度を水平付近に保つことが必須（後述）。

---

### 原因B: 試行1（ランダムターン）が y=0 脱出を引き起こした

**問題:** ランダム ±60°〜120° のターンでは上向きになる場合があり、地上アリが y=0 まで飛び出す。

地上（y < GROUND_LEVEL = 40）では `handleFreeMovement` 内の angle 補正が弱く（後述）、上向き角度を持ったアリは高速で y=0 に到達する。

**修正方針:** `avoidObstacle` の水平リダイレクトは「水平付近への強制リセット」として維持し、ランダム性は左右方向のみに持たせる。

---

### 原因C: 地上アリへの `wanderAngle` 不適用（最大の構造的バグ）

**該当箇所:** `Ant.ts` `handleFreeMovement()` 旧コード

```ts
if (this.y >= GROUND_LEVEL) {
  this.angle += wrapAngle(this.wanderAngle - this.angle) * (hasDirt ? 0.12 : 0.03);
} else {
  this.angle += (Math.random() - 0.5) * 0.2;  // ← 地上では wanderAngle を無視してランダムジッターのみ
}
```

地上アリ（y < 40）は `wanderAngle` による誘導を受けず、角度がランダムに漂流する。
上向き角度（angle ≈ -π/2）を持ったアリは速度 0.7/ステップで y=0 まで直進してしまう。

**修正方針:** 条件を削除して全アリに `wanderAngle` 補正を適用する。

---

### 原因D: 土持ちアリの `wanderAngle` が地上で上向きのまま

**該当箇所1:** `handleFreeMovement()` wanderAngle 設定ロジック

```ts
if (this.hasDirt) {
  this.wanderAngle = -Math.PI / 2 + ...;  // 常に上向き
}
```

地下では「地表へ向かう」ために -π/2（上向き）は正しい。しかし地上に出た後も -π/2 のままになり、原因Cの不適用が解消された後は積極的に y=0 へ引き戻される。

**該当箇所2:** `updateDigging()` 掘削完了時の初期化

```ts
this.wanderAngle = -Math.PI / 2;  // 常に上向きで固定
this.wanderTimer = 100;
```

掘削完了後 100 ステップの間、`wanderAngle = -π/2` が適用され続ける。地上で掘削が完了した場合も上向きになる。

**修正方針:**
- `handleFreeMovement` の wanderAngle 設定: `y >= GROUND_LEVEL` なら -π/2、地上なら +π/2 に分岐。
- `updateDigging` の wanderAngle 初期化: `y >= GROUND_LEVEL` なら -π/2、地上なら +π/2 に分岐。

---

### 原因E: 土の山（dirt mound）が地上空間を上方向に埋め尽くす（未解決の可能性）

**`dropDirt()` の動作:** アリの現在 y から下方向にスキャンし、type 2（ルーズダート）または type 3（保護ゾーン）の直上に土を積む。

地上（y < GROUND_LEVEL = 40）でアリが土を落とすと、土の山は y=40 から y=0 方向に成長する。多数のアリが地上に滞留すると山が y=0 まで達し、全アリが山に閉じ込められる。

**現状:** 原因C・Dを修正後にこの問題が残るかどうかは未検証。
